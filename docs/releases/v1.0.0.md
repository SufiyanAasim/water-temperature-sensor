# Water Temperature Sensor — v1.0.0

> *Codename: Ion*

---

## 📖 Overview

**Ion** is the first stable release of **Water Temperature Sensor**, transforming the [Atom](v0.0.1.md) proof-of-concept into a device that can be trusted to run unattended.

Where Atom proved the idea, Ion hardens every path around it: the firmware is now fully non-blocking, survives sensor faults without false alarms, verifies its own alarm hardware at boot, and streams structured CSV telemetry over serial. The display no longer flickers, the alarm no longer chatters at the threshold, and a session's temperature extremes are always one glance away in the log.

Ion also establishes the project's full repository standard — complete documentation, a project report, wiring guides, troubleshooting, issue templates, and compile CI for both Uno and Nano targets.

---

## 🎯 Objectives

This release was developed with the following goals:

* Eliminate every blocking call from the main loop.
* Make sensor failure impossible to mistake for a real reading.
* Guarantee the alarm hardware works before it is ever needed.
* Stop alarm chatter at the threshold boundary.
* Provide machine-readable telemetry for logging and plotting.
* Bring the repository to full documentation and CI standard.

## ✨ Added

### 🔔 Buzzer Self-Test

The device now chirps twice at boot.

A safety device whose alarm silently died is worse than no device at all — the self-test surfaces a dead or miswired buzzer the moment power is applied, instead of during a real over-temperature event.

---

### 📊 Session Min/Max Tracking

Every valid reading updates the session's lowest and highest temperatures since power-up, and both values are appended to every serial log row.

Any captured log now ends with the session's extremes — no post-processing, no scanning thousands of rows.

---

### 🛡️ Sensor Fault Handling

A disconnected or failed DS18B20 previously displayed **-127.00 C** as if it were a real reading.

Ion detects `DEVICE_DISCONNECTED_C` and instead:

* Shows `Sensor Error!` on the LCD
* Logs an `error` row over serial
* Forces the alarm off — bad data can never trigger a false alert

Startup additionally checks the bus and reports `No sensor found!` if empty.

---

### 🚨 Alarm Hysteresis

The alarm arms at **30 °C** but disarms only below **28 °C**.

Water hovering at exactly the threshold previously toggled the buzzer every second. The 2 °C gap makes the alarm decisive: once it sounds, it stays sounding until the water has genuinely cooled.

---

### 📤 Serial CSV Telemetry

The serial monitor (9600 baud) now streams structured rows:

```text
millis,temp_c,status,session_min_c,session_max_c
1023,25.50,normal,25.44,25.56
2047,30.12,alert,25.44,30.12
```

Ready for a spreadsheet, the Arduino Serial Plotter, or any downstream logger. Atom initialized serial but never printed a byte.

---

### 🌡️ Display Polish

* Custom `°` glyph — the display reads `25.50°C`, not `25.50 C`
* 12-bit resolution explicitly configured (0.0625 °C steps)

---

## 🔄 Changed

### ⚡ Non-Blocking Loop

`delay(1000)` and the library's blocking conversion wait are gone.

Conversions now run asynchronously (`setWaitForConversion(false)`) and all timing derives from `millis()`. The loop is never stalled by the sensor's ~750 ms conversion, which is what makes the pulsed alarm — and any future feature — possible.

---

### 🖥️ Flicker-Free Display

Atom called `lcd.clear()` every second, blanking the screen before each redraw.

Ion overwrites both rows in place, padding to the full 16 columns so shorter values completely replace longer ones. The display is now rock-steady.

---

### 🔊 Pulsed Alarm

The alarm beeps on a 500 ms cycle instead of holding a solid tone — harder to tune out, easier to live near.

---

### ⚙️ Configuration Block

All tunables are named `#define`s at the top of the sketch (`ALARM_ON_C`, `ALARM_OFF_C`, `READ_INTERVAL_MS`, `BEEP_PERIOD_MS`, `LCD_ADDRESS`, `SENSOR_BITS`), each documented in the README's configuration table. String literals moved to flash with `F()` to save SRAM.

## 💻 Compatibility

| Platform | Status |
|---|---|
| Arduino Uno | ✅ Verified (hardware + Tinkercad) |
| Arduino Nano | 🟢 Supported (CI-compiled) |

**Libraries:** OneWire · DallasTemperature · LiquidCrystal I2C

**Hardware Requirements:**

* DS18B20 on D2 with 4.7 kΩ pull-up
* I2C LCD @ `0x27` (A4/A5)
* Active buzzer on D3

## 🏁 Release Summary

**Ion** turns a working prototype into a dependable instrument.
