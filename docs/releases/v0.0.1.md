# Water Temperature Sensor — v0.0.1

> *Codename: Atom* · **Pre-release** · 2026-07-15

---

## 📖 Overview

**Atom** is the original proof-of-concept of **Water Temperature Sensor** — the smallest possible build that proves the idea works end-to-end: a DS18B20 probe in water, a live reading on a 16×2 LCD, and a buzzer that sounds when the water gets too hot.

It was verified first in Tinkercad simulation and then on a physical Arduino Uno breadboard build (see the [Wiring Guide](../hardware/Wiring.md#breadboard-build) for photos of both states).

As a pre-release, Atom favours simplicity over robustness — it is the baseline every later release measures itself against.

---

## ✨ Added

### 🌡️ Temperature Reading
- DS18B20 probe read once per second over OneWire
- Reading displayed on a 16×2 I2C LCD (`Temp: 28.63 C`)

### 🚨 Threshold Alarm
- Fixed 30 °C threshold
- Buzzer held on continuously while above threshold
- LCD status line: `Temp Normal` / `ALERT: Temp High!`

## ⚠️ Known Limitations (addressed in v1.0.0 "Ion")

| Limitation | Consequence |
|---|---|
| Blocking `delay(1000)` loop | Everything stalls ~1.75 s per cycle (delay + conversion wait) |
| No sensor fault handling | A disconnected probe displays `-127.00 C` as if it were real |
| No hysteresis | Buzzer chatters on/off when water hovers at exactly 30 °C |
| `lcd.clear()` every second | Visible display flicker on each refresh |
| Serial initialized but unused | No way to log or plot readings |
| Solid buzzer tone | Alarm is grating and easy to tune out |
